# Project Summary & Reflection

## What I Built

A complete patient vital signs monitoring simulator that mimics real medical 
device software used in hospitals. The project simulates heart rate, SpO2, 
and blood pressure data, displays it on a live dashboard, runs automated 
tests, and uses machine learning to detect dangerous readings.

## Why I Built This

As a Medical Systems Engineering student actively applying to medical device 
companies in Germany, I wanted to demonstrate practical skills relevant to 
the industry — particularly after a interview opportunity with a 
medical device company that focused on test automation and data processing.

## What I Learned

- **Python fundamentals**: functions, loops, conditionals, data structures
- **Git & GitHub**: version control, committing, pushing code professionally
- **Test Automation**: writing Pytest test suites, understanding why 
  medical software requires rigorous testing
- **Web Dashboards**: building live, interactive interfaces with Streamlit
- **Machine Learning basics**: training an Isolation Forest model for 
  anomaly detection, understanding train/predict workflows
- **Data Visualization**: creating professional charts with Matplotlib
- **Project Structure**: organizing a Python project the way real 
  companies do (requirements.txt, .gitignore, README, tests folder)

## Challenges I Faced

- Initially struggled with terminal/command line navigation
- Had Python version conflicts (3.11 vs 3.12) that required learning 
  how Python environments work
- Understanding why automated testing matters, not just how to write it
- Connecting multiple Python files together (imports between modules)

## What I Would Add Next

- Connect to real sensor hardware (Arduino + actual heart rate sensor)
- Deploy the dashboard online so it's accessible without running locally
- Add a database (SQL) instead of CSV files for proper data storage
- Build a more sophisticated ML model with labeled real patient data
- Add user authentication for multi-patient monitoring

## Reflection

This project taught me that building software for medical devices requires 
a different mindset than regular software — reliability and testing aren't 
optional, they're critical because real patient safety depends on it. 
I now understand why companies  emphasize test automation so 
heavily in their hiring process.
